# VPS

核心目的不是「搭梯子」，而是拥有一块可长期持有、可自我叙述、可承载未来 Agent 的个人计算与网络延伸——出口主权、运维记忆、服务编排能力比单点功能更重要。

当前目的明确、基础设计清晰，最大断裂点是：哲学架构（单一控制面 + 个人节点）已在文档闭合，但过渡态尚未收束为稳定日常。

## 已完成

- 确立「个人网络节点」而非「租一台服务器」的定位：接受物理延迟约束，用协议选型与本地分流补偿地理劣势
- 完成「单一控制面」认识闭环：日常流量经统一管理入口；运维踩坑固化为可复用流程
- VPS 侧安全与出口层就绪：密钥登录、入侵防护、防火墙；双协议出口与 BBR；密钥轮换完成
- 服务基底已铺设：容器 + 反向代理 + HTTPS 验收；监控 hub；备份脚本与恢复演练已在 VPS 侧验证
- 文档/运行双层分工清晰：运行层与认知层分目录维护，本地 Git 已初始化并排除私密目录

## 下一步

1. 完成监控 agent 配对，闭合「可观测性」断裂点（hub 已部署，agent 未配对）
2. 主出口稳定满窗口后退役旧通道，消除与主 TUN 路径的路由/DNS 互斥风险
3. 完成本机异地备份拉取验证，并刷新状态文档快照日期，消除文档层与运行层滞后

## 遗留问题

- 纯基础设施 + 文档仓库，无可执行应用代码树
- 部分总览文档与当前状态快照存在认知漂移
- 运行配置与认知文档分属双目录，需手动同步
- 观测 agent / 本机备份拉取 / 旧通道停用三项尚未关环
