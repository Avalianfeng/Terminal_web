# 认知协作

核心目的不是「又一个管理得更好的论坛」，而是探索语言能否拥有 Git 那样的可操作性——让思想可分叉、可 diff、可重新激活（Reactivation），使认知成为像代码一样可以协作演化的对象。

当前认识与目的已高度清晰（哲学收敛 ✅、Language OS 转向 ✅），最大断裂点是：设想已从「人与人的讨论」编译到「语言即数据结构」，但最底层 Schema 与一次真实 diff 尚未存在。

## 已完成

- 完成第一轮哲学压力测试：定位从「统一真理机器」收敛为「认知演化基础设施」——保留分歧、允许多路径并行、AI 只做结构整理不做裁判
- 确立系统内核对象与操作模型：核心单元是思想节点（Thought Node）；四种基本操作 Create / Respond / Fork / Link
- 完成对外宣言与构建者视角的阶段性叙事，明确要验证的不是技术而是「是否有人愿把共同理解世界当作长期投入」
- 关键转向：从「认知协作平台」推进到「语言操作系统（Language OS）」——管理对象变为 Claim / Premise / Evidence / Scope 等可 diff 的语言结构层
- 建立文档演化体系：STATUS 索引、文档头规范、参考研究摘录，并厘清与竞品差异及「强制关联而非禁止发布」的设计方向

## 下一步

1. 撰写思想节点双层 Schema 规格（人类可读层 + 机器可读层），规定 Fork / Diff / Link 的字段与边界
2. 实现最小验证脚本：两段自然语言 → 结构层提取 → 结构化 diff，验证「语言能否像 Git 一样被部分形式化」
3. 做极简静态 landing（manifesto 摘要 + Fork 概念图 + 四操作说明），并整理工作区文档断裂

## 遗留问题

- 无可执行原型；技术路线（向量库、Argument Mining、首发权权力结构）仍开放
- 工作区文档存在断裂与未入库材料
- 无 landing page / UI 草图；无单元测试与 CI
