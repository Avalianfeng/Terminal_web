# bookchat

核心目的不是「又一个带 AI 的电子书阅读器」，而是探索阅读能否拥有对话的节奏感——段落按自己的步速展开，理解发生在需要时才调用外部认知，且用户始终掌握上下文边界与成本。

当前目的与分层设计已清晰（设计文档 ✅、L0–L3 闭环 ✅），最大断裂点是：哲学层的「成本可见、上下文可审」已在 UI 搭好骨架，但全书预估与实时预览两处尚未编译完毕。

## 已完成

- 确立核心认识论：阅读不是「问答机器人」，而是以聊天节奏逐段呈现结构化文本——讲述本身零 AI
- 完成 EPUB → 段落序列 → 对话式推进全链路：解压、目录解析、可配置 Chunker、四级 AI 分层（L0–L3）
- 搭建认知透明度基础设施：统一 context 构建、Context Panel、Token Observatory，让用户看见 AI「看到了什么」
- 实现本地-first 持久化：进度恢复、EPUB 哈希去重、「记住此书」、单文件离线构建
- 完成可定制阅读体验壳层：中英 i18n、bubble/stream 双模式、章节/提问侧栏、多提供商流式接入

## 下一步

1. 闭合「预估读完全书」：基于进度、AI 层级与提问频率估算剩余 token/费用
2. 修复 Context Panel 预览与状态脱节，使「AI 看见什么」始终与当下阅读位置同步
3. 为 chunker 与 ai-protocol 补最小回归测试

## 遗留问题

- Vanilla JS + Vite 架构；API Key 明文存 localStorage、浏览器直连 API
- 定价硬编码且设置 UI 不可编辑；错误处理无统一边界
- 长会话全量重绘可能卡顿；测试与版本历史空白
