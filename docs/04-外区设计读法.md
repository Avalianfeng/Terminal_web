# 外区设计读法 + 禁区清单

> 状态：约束已定；**质感已解冻并完成一轮抛光**；Phase 1/2a/2b 开合与 demote 已落地  
> 范围：层 A（页面工作区）+ 层 C（阅读表面）。终端表面另论，不靠外区「变好看」来补。  
> 相关：`01-v0.2-工作区图景.md`、`05-v0.3-档案与命令.md`

---

## 1. 一句话读法

**访客打开档案时，像在阅览室摊开一份文件；终端仍是探索器，不是被页面装饰挤掉的工具栏。**

Reading this as：个人数字档案的**阅读场**（editorial / archive），给陌生访客读「一个人的材料」，不是招聘向作品集，也不是动效展示站。

---

## 2. 类似但不一样（参考锚点）

参考：[Rajat Berry · Wall of Portfolios](https://www.wallofportfolios.in/portfolios/rajat-berry/) 一类——强调技术与设计呈现、作品/案例驱动、个人品牌展示。

| | 那一类站 | cylf.me 外区 |
|--|----------|--------------|
| 主语 | 设计师 / 作品 | 档案对象 / 文本与时间 |
| 成功标准 | 作品看起来厉害、可被雇佣 | 打开后愿意把一篇读完、还想再 `open` |
| 视觉重心 | 项目封面、案例叙事、Craft 秀场 | 纸面、标题层级、元信息、与终端的 Dual Phase |
| 动效角色 | 常为「展示设计能力」 | 只服务开合与焦点，不为秀技 |
| 技术感从哪来 | UI 本身很「设计」 | **终端层**已承担；外区应偏纸与字，不跟终端抢「酷」 |

可学：克制的排版纪律、清晰层级、少而准的交互。  
不学：作品网格英雄区、案例页模板、为证明设计力而堆的动效与组件。

## 3. 三个旋钮（外区默认）

| 旋钮 | 取值 | 含义 |
|------|------|------|
| 变化度 | 4–5 | 稳、档案感；不对称为辅，不为怪而怪 |
| 动效 | 2–3 | 面板出现/关闭、焦点回终端；全站签名动效 ≤ 3 |
| 密度 | 3 | 阅读向：字与留白优先，少装饰块 |

---

## 4. 硬约束（改 UI 前对照）

1. **Dual Phase 不可破**：浅色外区衬深色终端；阅读面是纸，不是第二块黑壳。  
2. **字体**：纸面正文（衬线）+ 终端/路径等宽，至多两套；不用 Inter/Roboto/系统默认堆出「通用落地页」。  
3. **强调色 ≤ 1**（可随 palette 变，但同时只有一条强调逻辑）。  
4. **布局**：终端在上、阅读在下（`stack`）；阅读高度随内容，页面滚动；不与终端争视口做常数对半分。  
5. **窄屏必过**：≤960px 下 Dual Phase 仍成立；安全区 padding；关闭/rail 触控可用；不做第二套手机 IA。  
6. **组件库**：不把 React Bits / Aceternity / Magic UI 等当外区底座；灵感可看，落地自写或抠改到认不出货源，且一页特效块 ≤ 1。  
7. **动效预算**：只服务「打开可读 / 关闭回探索」；遵守 `prefers-reduced-motion`。  
8. **区块优先于动作**：先定主阅读槽 / rail，再套进退场；禁止未定终点就做「从终端涌出 + 推终端」整套秀场。

---

## 5. 禁区清单（出现即视为偏航）

- 紫/蓝渐变英雄区、玻璃拟态铺满、mesh/aurora 当主视觉  
- 等宽三列功能卡、图标墙、stat 数字条、「Get Started」式 CTA 群  
- 满页 fade-in / 磁吸光标 / 粒子背景 / 自定义炫光光标  
- 把阅读面板做成又一张「酷卡片秀」，压过正文  
- 外区再做一套深色终端皮肤，取消纸面对比  
- 为「更像作品集」加项目封面墙、技能条、假数据指标  
- 未先改本文档约束就引入整库动效组件  
- 现阶段做「从终端内部涌出 + 改终端几何推上去 + 右侧缩略多窗」整包（过度设计，暂缓）  
- 完整手机 App 化（底部 Tab、隐藏终端、单独手机路由）

## 6. 外区事项

### 质感（已解冻 · 一轮抛光完成）

1. ~~字体 Dual Phase~~ — `Source Serif 4`（纸面）+ `JetBrains Mono`（终端/路径）；中文回落系统宋体栈。  
2. ~~纸面层级~~ — chrome / prose / timeline 收拢为 CSS；行宽约 66ch。  
3. ~~工作场~~ — 梯度与网格只衬窗口；舞台 `safe-area` padding。  
4. ~~窄屏必过~~ — 终端高度压低；rail 横条触控；关闭钮命中区扩大。

### Phase 1（已完成）— 单主槽开合

- 主槽进/退场：展开 + 淡入淡出；关闭时与滚回终端并行。  
- 滚动：`block: start` 把阅读区顶上来；`focus({ preventScroll })` 不抢滚动。

### Phase 2a（已落地）— `main` + `rail` 区块

- 状态：`ReadingState { main, rail[] }`；rail 硬顶 8。  
- 布局：桌面右侧窄列；窄屏主槽下方横向条。

### Phase 2b（已落地）— demote 动作

- 旧 main → rail：幽灵层短 scale + 位移；与新主槽进场并行。  
- 「从终端涌出」仍为可选润色，默认不排期。
