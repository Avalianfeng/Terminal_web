# 大计划纲要与最小 v1.0

> 状态：纲要已定；**V0（纯文本控制台阶段）视为结束**；正在进入最小 v1.0  
> 相关：`01`–`05` 为 V0 过程文档，约束仍有效；冲突时以本文版本分界为准  
> 目的：收束「后面还要什么」与「v1.0 只许做什么」，避免入口/皮肤/媒体一哄而上

---

## 1. 版本分界

| 阶段 | 一句话 | 结论 |
|------|--------|------|
| **V0** | 证明：人物档案可用**终端**探索，正文可在**纸面**阅读 | **已结束** |
| **v1.0** | 从「能用的控制台」变成「可被人或 Agent 认真使用的档案站」 | **当前进入** |
| **v1.x / v2** | 多媒体对象、更完整编辑 UI、更深 AI 工作流 | 大图景，不塞进最小 v1.0 |

**V0 已交付（不再当主线扩）**

- Dual Phase：浅色工作场 + 深色终端壳 + 外区阅读 `main`/`rail`
- xterm 档位 1、Tab、批量 `open` / `find` / `status`、palette、fullscreen
- 内容形态以 Markdown / JSON **纯文本**为主；命令 runtime 自有（不嵌 Linux）

**V0 明确不再优先**

- 轻量/完整进场叙事（等内容与机器接口都站稳再做）
- 再堆 `/themes` 皮肤、外区动效库、真窗管

## 2. 产品句（v1 起）

**cylf.me 是策月帘风的个人数字档案：终端是探索器，纸面是阅读器，HTTP API 是给 Agent 的正式入口。**

- 对人：长文与世界观材料要在站内有好的呈现，而不是「markdown 预览凑合看」。
- 对 Agent：服务器在线即可读（约定范围内）并在鉴权下写——**不依赖本机 Cursor CLI 是否方便**。
- 对以后：音乐 / 图 / 视频 / 外站关联是档案对象的扩展，不是把站点改成媒体站首页。

## 3. 大图景（写入计划，但不等于现在全做）

### 3.1 呈现与对象

| 方向 | 含义 | 大致落点 |
|------|------|----------|
| 纯文本深化 | 自我探索 / 世界观长文的纸面与目录可读性 | **v1.0 核心** |
| 图片 | 附图、图注、与文档挂接 | v1.x |
| 音乐 | 可播对象或文档内嵌音频 | v1.x |
| 视频 | 外链或自托管片段 | v1.x / v2 |
| 外站关联 | 其他自有站点 / 项目入口作为档案边 | v1.x |
| 进场叙事 | 访客第一次理解「进了什么世界」 | 文本与 API 稳后 |

### 3.2 权限与编辑

| 方向 | 含义 | 大致落点 |
|------|------|----------|
| 身份 | 访客只读；所有者（及授权 Agent）可写 | **v1.0 最小闭环** |
| 多入口编辑 | 终端命令 / 阅读面板 / API 同一写路径 | v1.0 先 API（+可选极简 UI）；完整 UI 在 v1.x |
| 安全模型 | token / session、范围权限、审计；禁止「能改无鉴权」 | **v1.0 起硬约束** |

### 3.3 AI 获取与深层操作

| 方向 | 含义 | 大致落点 |
|------|------|----------|
| **机器可读拉取** | 开放约定接口：索引、单篇正文、元数据 | **v1.0 必做** |
| **安全写回** | Agent 经鉴权创建/更新档案对象（先文本） | **v1.0 最小写** |
| 站内 AI 深操作 | 对话式发掘、批注、多文推理 UI | v1.x / v2 |
| Cursor CLI | 仅作开发者可选工具；**不是**生产依赖 | 始终非必须 |

**AI 入口原则（钉死）**

1. **Server-online = operable**：Agent 只依赖站点（或同机对外服务）在线与凭证，不依赖某台电脑开着 Cursor。  
2. **读与写分离**：公开读可以宽；写必须鉴权 + 范围（例如只许 `thoughts/`）。  
3. **API 是一等公民**：终端与纸面是人的视图；Agent 不爬 HTML，走正式 JSON/Markdown 接口。  
4. **编辑经同一真相源**：人 UI 与 Agent 写的是同一内容层，避免「CLI 改盘、站点另一份」双源。

## 4. 最小 v1.0（只许这些）

> 目标：结束「纯控制台 demo 感」，让**长文可读**、**Agent 可拉可写（安全）**。  
> 非目标：多媒体齐全、完美编辑器、进场动画、AI 聊天产品。

### 4.1 必做

| # | 项 | 完成样子 |
|---|----|----------|
| A | **纯文本呈现** | 阅读面板适合长文（层级、元信息、路径感）；`projects`/`thoughts` 列表带摘要级可读信息；中文文案为主路径 |
| B | **公开读 API** | 至少：档案索引 + 按 slug/path 取正文（及必要元数据）；稳定、可文档化、可被脚本/`curl`/Agent 直接用 |
| C | **身份 + 安全写 API** | 所有者（及可选 Agent token）可创建/更新文本对象；无凭证不可写；写回进入与站点相同的内容源 |
| D | **契约文档** | 一篇给人看的「Agent / API 用法」（鉴权、范围、例请求）；README 指向 v1.0 |

### 4.2 明确不做（v1.0）

- 音乐 / 视频播放器、图库产品化
- 完整站内富文本编辑器、多入口精修 UI（API 写优先即可）
- 进场 `whoami` 剧本、第二套非终端首页
- 「无鉴权的方便写入」、把 Cursor CLI 写成唯一编辑方式
- 多人档案、复杂 RBAC（v1.0 只需：公开读 / 持钥写）

### 4.3 成功标准

- 冷访客：能用终端找到并读完一篇世界观/自我探索长文，纸面不别扭。  
- Agent：仅凭「服务在线 + 文档中的读接口」能拉取文章；持正确凭证能更新一篇并在站点刷新可见。  
- 开发者：不打开 Cursor 也能用 HTTP 完成读/写验收。

## 5. 建议实现顺序（v1.0 内）

1. **文本呈现**（A）— 与现有 Dual Phase 连续，立刻提升「纯文本项目」的实用性  
2. **读 API + 用法文档**（B + D 读半）— Agent 可读，先建立机器入口  
3. **鉴权写 API**（C + D 写半）— 最小 token；范围默认收紧  
4. 用 1–2 篇真长文验收 A；用脚本验收 B/C  

进场叙事、多媒体、深 AI UI → 全部排到 **v1.0 完成之后**。
