## 2. C0 约定（已落地）

### 外区

- 浅亮阅读场（衬出窗口边界），**不再**整页等同黑终端。
- 关键词：**Surface**（面）、**Elevation**（抬升：谁在谁上面）。

### 终端

- 仍是主体交互；打开文档后**仍可输入命令**。
- Prompt 随 cwd：`visitor@archive:~$` / `~/projects$`。
- Tab 补全：命令名、路径、`open`/`cat` 的 slug（prefix → LCP → cycle）。
- 表面与运行时分离：表面可接 `xterm.js`；命令仍走 Archive runtime。**不嵌 Linux。**

### 阅读面板（C0）

- `open` / `cat`（文档）/ `timeline` → 外区独立面板；`open timeline|person` 可用。
- 布局模式：`layout: floating`（浮层）。
- 关闭：× / `Esc`；`clear` 同时关面板。
- **未做**：贴左、贴下、最小化、多窗、拖拽。

### 动效

- 不做首次进场 `whoami` 动画。
- 部件级：输出短间隔、面板进出。
- 遵守 `prefers-reduced-motion`。

## 5. 延后：身份系统 + 多入口编辑

> 只定方向，本阶段不实现。

### 原则

- **编辑与身份绑定**：无身份则只有公开只读；有所有者身份才能写。
- **多入口编辑**：同一档案对象可从多处进入编辑，而不是只做一个隐藏后台。

### 可能入口（未来）

| 入口 | 说明 |
|------|------|
| 终端命令 | 如 `edit <slug>` / `append thoughts/...`（仍走 runtime） |
| 阅读面板 | 所有者可见「编辑」→ 面板内或旁路编辑态 |
| 内容文件 / 将来 API | `content/` 或远端存储的正式源；终端与面板都是视图 |

### 不提前做的原因

- 写路径会强迫存储、冲突、权限模型定稿；现在内容形态未稳。
- 半套「能改但不能鉴权」比只读更危险。

### 与 xterm / AI 的关系

- 身份就绪后，AI/脚本批量命令才有「以谁身份执行」的主语。
- 在此之前，终端表面可先加强只读探索与流式**展示**。
